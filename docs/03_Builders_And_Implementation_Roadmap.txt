03_Builders_And_Implementation_Roadmap.txt

Purpose

This is the active roadmap for Character Builder, Summoning Circle, Forge, and
remaining app work. It consolidates useful current roadmap content and removes
stale step-by-step prompt chains.

This document is implementation roadmap authority, not combat rule authority or
final costing law. Combat rules live in 01; power grammar and costing live in
02.

ROLEPLAY BUILDER AUTHORITY: docs/roleplay-standard-library.md locks the
compositional flow. Builder order is Name, Theme, Intention, Method, Outcome
Contract, Scope, Impact, Dice, Counter, Restriction. Outcome Contracts filter by
Intention and Method only. A selected contract then supplies only completed
Scopes and Impacts, with stable-order fallbacks that retain the family. Exact
tuple variants and tuple privilege keys are superseded implementation history.
Current coverage is 88 planned / 88 completed / 0 missing cells. Persuasion
filters to Appeal, Rally, and Steel Yourself. Steel Yourself exposes Sustain
Personal Resolve, Self only, all four Impacts, Help, and no Counter. Existing
registry selection moves the default One Target Scope to Self and renders
`Roll X dice.` without a Builder page fork. Declared Personal Course remains
runtime-only.


Current Builder Surfaces

Summoning Circle:

- First live implementation surface for the shared power architecture.
- Uses the shared power core in first-pass form.
- Supports practical chassis selection, packet-local authoring, Counter,
  Commitment Modifier, Lifespan where relevant, timing-gated descriptor
  rendering, and power-cost integration.
- Remains the reference authoring surface for shared power legality.

Character Builder:

- Reuses the shared power builder direction rather than a character-only fork.
- Stores player-authored powers in CampaignCharacter.builderData.
- Uses the shared power descriptor renderer and canonical Phase 6 cost resolver.
- Uses a player-facing Power Point pool: Character Level x 50.
- Converts BasePowerValue to player Power Point spend through the current
  player power spend scalar.
- Has current runtime support for Signature Move with a separate pool:
  Character Level x 20.
- Signature Move should remain one authored move, use the same power builder
  rules, and render at the top of the Power Sheet.
- Equipment and Backpack flows are campaign-local and must hydrate current item
  details rather than copying stale item data into the character.

Forge:

- Forge creates and renders campaign-local items.
- Forge output work should compare item output by table-facing profile, not raw
  backend fields or source origin.
- Forge implementation should not be rewritten merely to support Character
  Builder. Reuse descriptor and output helpers where possible.


Current Complete Or Substantially Complete Areas

- Shared power foundation phases 1 through 5 are complete enough to close as
  design phases.
- Summoning Circle shared power implementation is substantially complete.
- Monster power builder has a minimal but real implementation.

Remaining Work

- Keep Character Builder and Summoning Circle power legality in sync through
  shared pure helpers where practical.
- Continue validating Character Builder save/load, budget validation, preview,
  and print parity.
- Continue removing duplicated or stale power-builder logic between Summoning
  Circle and Character Builder.
- Preserve Packet 1 as the sole defence gate.
- Keep secondary packet dice ownership out of Character Builder V1 unless a
  later rule changes it.
- Keep packet duration turns capped according to current validation rules.
- Continue real-monster and real-character validation after mechanical changes.
- Build printable power reference output and effect-slip output when scheduled.
- Add runtime active-effect model and optional encounter runner later.
- Build future Setting Package policy and tooling after campaign-local workflows
  are stable.

  reusable purchased privilege, derived lane/outcome, compatible authoring,
  Counter eligibility, and eventual costing/cooldown pressure.
- Scene Manifestation is narrated by the Game Director from current context.
  "Method defines the approach. Theme personalises the approach. The Outcome
  Contract defines the result."
- Remove free-text Success Outcome from standard contract authoring.
- Initial standard registry only:
  - HIDE_FROM_IMMEDIATE_DANGER: Help; Intervention / Rescue / Minor / One
    Target or Small Group; Counter ineligible.
  - SECURE_IMMEDIATE_SAFETY: Help; Intervention / Rescue / Standard / One

  - DENY_IMMINENT_HOSTILE_ACT: Hinder; Intervention / Interrupt / Major / One
    Target; Counter eligible.
  - DIVERT_IMMEDIATE_ATTENTION: Hinder; Deception / Distract / Minor / One
    Target or Small Group; Counter ineligible.
- Filter standard contracts by Intention, Method, Scene Impact, and Scope.
  Keep Custom Outcome - GD Review Required available for all combinations.
- Custom Outcome stores a proposed lane and request, remains Pending GD
  Approval, cannot be automatically costed, and has pending final Cooldown.
- Powers calculate quantified effects. Roleplay Abilities establish outcomes;
  they do not convert into wounds, Block, Dodge, Resist, stacks, units,
  measured movement, or other quantified Power mechanics.
- Roleplay Abilities v0 do not author Ability Shape or Duration.
- Ability Shape is removed from v0 because Sustained, Prepared, and Triggered
  are deferred / removed, and a lone Immediate shape would be clutter.
- Standard Use in combat consumes the user's Power Action. Out-of-combat use is
  an ordinary scene declaration with no combat action cost.
- Roleplay Window use costs 1 Response, is universal to Roleplay Abilities, and

  declared in each Game Director-opened Roleplay Window.
- Roleplay Abilities cannot be used as normal Assists. Preserve ordinary Power

- Counter uses 1 Response and requires the actor to be personally targeted by a
  hostile action / effect plus authored paid Counter permission. Counter adds

- Attribute should not be a builder field.
- Builder should not suggest typical Attributes.
- The Game Director validates the selected contract, derived lane/outcome,
  Narrative Theme, Method, Scope, Scene Impact, fictional position, and
  Restriction,
  then declares Difficulty and Attribute. On success, no additional roll or
  target action is required to establish the contract outcome.
- Roleplay numeric costing implementation is BLOCKED / PARKED pending every

- Do not add a Roleplay resolver, tuning rows, provisional values, Builder

- Future implementation requires one pure resolver receiving normalized

  fetches, or built-in numeric fallback.
- Deterministic smoke and benchmark boundaries must cover all live variants,

  evidence before any numeric Builder surface is considered.
- The complete non-numeric implementation contract and unlock gates are in
  [Roleplay Ability Costing Readiness](roleplay-ability-costing-readiness.md).
- Avoid warning language implying roleplay-heavy builds are suboptimal.
- For the first slice, test-card representation may be used temporarily where

Campaign-Local Asset Model

At the current development stage, Characters, Monsters, and Forge Items are
campaign-local.

Character Builder should equip from the character's Backpack, not from every
campaign item.

Equipped selections should be slot-based references to assigned Backpack rows.
The Builder should hydrate current item details from Party Inventory / Forge item
templates and sanitize slot selections against current Backpack quantities on
load/save.

Backpack items do not affect derived combat stats unless equipped.

Future Setting Packages may introduce global protected source assets. When that
happens, active/global package assets require Architect approval before mutation.
Campaign-local clones remain editable inside approved balance-test campaigns.


Source Mutation Policy Summary

Codex may create and edit Characters, Monsters, and Forge Items only inside an
approved balance-testing scope.

Inside the dedicated balance campaign, campaign-local calibration characters,
monsters, Forge items, temporary variants, cloned test assets, and
simulation-only overrides may be edited for calibration.

Codex may not freely alter:

- assets marked baseline
- assets marked canonical
- assets marked locked
- accepted calibration anchors
- demo/playtest campaign assets outside the balance-test scope
- any campaign or asset explicitly reserved by the Architect

If an asset is used as a comparison baseline, clone it before mutation.


Roleplay Outcome Contract Variant Builder Rules

Standard Outcome Contract definitions may contain multiple explicit variants,
matched against Intention, Method, Scene Impact, and Scope. The builder must
resolve Success Outcome, Counter eligibility, and future privilege-cost
adjustment from the exact matching variant; it must not store derived lane or
outcome prose on the Ability and must not scale prose generically.

DRAW_HOSTILE_ATTENTION remains one dropdown contract ID and player-facing entry,
Draw Hostile Attention, across its Intimidation / Challenge / One Target Minor,
Standard, Major, and Legendary variants. Changing between those four Impacts
keeps the ID and immediately changes the derived exact Success Outcome. Changing
Scope, Intention, or Method outside the family clears the selection and
Counter. Counter stays disabled at every tier. The derived panel should identify
the current Impact variant; no separate duration field is authored.

The current visible control labelled "Additional Restriction Type", its band,
tag, and free-text fields are prototype UI and migration input, not the
permanent editor contract. The future shared editor uses exactly Standard
Structured, Campaign-Custom Structured, and Custom Narrative modes, renders a
separate Restriction descriptor, and never rewrites ordinary target or Outcome
Contract grammar. Legacy `restrictionType`, `restrictionBand`,
`restrictionTag`, and `restrictionText` names must be read conservatively during
migration but are not the permanent persisted API.

The standard Method registry contains Appeal and Rally under Persuasion,
Misdirect and Distract under Deception, Rescue and Interrupt under Intervention, Challenge
and Overawe under Intimidation, and Discern Truth under Perception. All five Intentions have
an approved standard Method. Rally unites a bounded group around one shared
course through common purpose, courage, duty, identity, hope, urgency, or mutual
reliance; it remains distinct from one-target Appeal, threat/fear-led
Intimidation, and false/misleading Deception. Misdirect covers direct falsehood, omission,
implication, distraction that establishes a false interpretation, selective
truth, staged cues, and deceptive framing. Its classification follows the
materially false conclusion deliberately created, not whether every literal
sentence is false; it remains distinct from honest Appeal, threat/fear-led
Intimidation, and attention-only distraction. A standard Method details panel
shows name, Intention, definition, legal approaches, and exclusions, with legal
approaches explicitly presented as examples rather than an exhaustive phrase
list.

Overawe uses credible threat, power, authority, reputation, certainty, omen, or
consequence to make continued opposition feel untenable. Challenge retains
Draw Hostile Attention and contests attention/hostility/rivalry; Overawe owns
Break Resolve and breaks one bounded course for one target or a Small Group.
Appeal uses values/reason and
Misdirect uses false or misleading conclusions; false threats remain Misdirect.
The Overawe details panel derives its exact eight illustrative approaches and
fourteen exclusions from the Method registry.

Custom Method - GD Review Required remains Pending GD Approval and exposes
Proposed Method Name plus a Custom Method Request for reusable approach,
applications, and boundaries. It offers no standard contract matching,
automatic costing, or trusted final Cooldown. Discern Truth filters to two
compatible standard Help contracts in registry order for Perception / One
Target at Minor, Standard, Major, and Legendary Impact:
UNCOVER_CONCEALED_TRUTH and REVEAL_EXPLOITABLE_WEAKNESS. All variants are
Counter-ineligible and derive exact registry-authored outcomes. No Spot
Weakness, Investigate, Search, Analyse, or synonymous Method is added: Discern
Truth already owns the reusable perceptive approach. Custom Outcome review remains available
for non-standard requests and an explicitly stored Custom Outcome is never
silently converted to the new standard family.

Reveal Exploitable Weakness is the accepted v0 working name and remains eligible
for the naming pass. Its exact variants and reserved keys are:

- Minor / REVEAL_EXPLOITABLE_WEAKNESS_MINOR: "you identify one small
  immediately useful weakness, opening, or opportunity concerning the target
  for the current meaningful exchange; if no qualifying exploitable
  opportunity exists, you learn that none is presently accessible"
- Standard / REVEAL_EXPLOITABLE_WEAKNESS_STANDARD: "you identify one useful
  exploitable weakness, route, pattern, dependency, or leverage point
  concerning the target for the current situation; if no qualifying
  exploitable opportunity exists, you learn that none is presently accessible"
- Major / REVEAL_EXPLOITABLE_WEAKNESS_MAJOR: "you reveal one central exploitable
  vulnerability or opportunity concerning the target that is shaping the
  current scene and can materially change how it is approached; if no
  qualifying exploitable opportunity exists, you learn that none is presently
  accessible"
- Legendary / REVEAL_EXPLOITABLE_WEAKNESS_LEGENDARY: "you reveal one defining
  vulnerability, hidden route, dependency, or leverage point concerning the
  target whose significance extends beyond the current scene; if no qualifying
  exploitable opportunity exists, you learn that none is presently accessible"

Target means the bounded subject to which a Roleplay contract applies. One
Target may be a character, creature, object, mechanism, bounded location or
structure, plan, ritual, environmental feature, or another coherent identified
scene subject when supported. A diffuse crowd, unrelated subjects, battlefield,
city, faction, army, war, organisation membership, or abstract collection does
not become One Target by naming it as one. The GD establishes boundary and
access before Attribute or Difficulty.

The accepted runtime Declared Aim binds the answer. A qualifying opportunity
concerns that target and Aim, is true or coherent, Impact-appropriate, reachable,
actionable, and not already understood in the same actionable form. It may be
established, inferred, an existing property, or a coherent new Scene
Manifestation. Lack of prepared material or plot inconvenience cannot justify
"none"; only genuine absence of an accessible qualifying opportunity can.
Uncover grants concealed information; Reveal grants actionable leverage. The
result informs later choices but neither automatically exploits it nor creates
or changes quantified mechanics.

The registry-driven Builder exposes both options without a page fork, retains
an existing selection while compatible, and never auto-selects or converts a
Custom Outcome. It adds no Recipient Type, target/subject, investigation,
Declared Aim/objective, weakness, or opportunity fields. The live inventory is
twelve Methods, fourteen standard families, fourteen family privilege keys, 88
planned cells, 88 completed cells, and no missing cells. Numeric
costing and cooldown implementation remain parked.

Rescue exposes HIDE_FROM_IMMEDIATE_DANGER and SECURE_IMMEDIATE_SAFETY at all
four Impacts for One Target or Small Group. Both are Help and
Counter-ineligible and remain in their stable families. The 43 outcomes and
descriptors that predate completion remain exact. Historical Small Group
regression examples include:

- HIDE_FROM_IMMEDIATE_DANGER_SMALL_GROUP: "every accepted member of the selected
  group becomes hidden from one declared immediate danger"
- SECURE_IMMEDIATE_SAFETY_SMALL_GROUP: "every accepted member of the selected
  group is secured from one declared immediate peril and is no longer directly
  threatened by it"

Self, Large Group, and Faction / Army remain unsupported. Rescue remains the Method; Extract, Catch, Pull Free,
Shield With Body, Stabilise, Escort, and Evacuate do not become standard
Methods.

Before Difficulty, the player declares fixed membership, one shared concrete
bounded current or imminent danger/peril, the exact Rescue contract, and a
plausible coherent Rescue/Theme means and result for every accepted member. The
GD confirms universal eligibility; one ineligible, separate, unreachable, or
incoherent member causes narrowing, revision, or rejection before Difficulty.
Success binds every accepted member, without a majority result, hidden
exemption, post-success narrowing, per-member resistance, or another Hide,
movement, defence, rescue, or target roll. The GD may narrate a coherent
previously unspecified route, cover, shelter, or safe feature, but cannot use
lack of a prepared detail to reduce success; genuinely impossible access is
rejected before Difficulty. Relocation is narrative Scene Manifestation, not
measured movement, and the player selects no exact square, distance, path,
formation, elevation, hostile movement, or separate destination.

The safe state persists according to selected Impact until voluntary re-entry,
decisive breach, or Narrative Resolution as applicable; the original source may
remain dangerous. Hide scales concealment persistence by Impact. Deny remains
Interrupt / One Target / Counter-eligible at all four Impacts. Secure changes
scene state only before formal hostile declaration and cannot be introduced
once the Response Window begins.

Membership fixes at Difficulty, does not propagate to later arrivals or wider
groups, and one member's later exposure need not end the others' result. The
registry-driven Builder exposes both Scopes automatically at the supported
Impact. One Target to Small Group or back retains the family while changing the
composed outcome and descriptor; the family key remains stable and Counter
stays false. Unsupported Method, Intention, or Scope clears selection/Counter;
Name, Theme, Dice Count,
and restrictions retain it. The Builder adds no danger/peril, group-member,
route, cover, shelter, destination, safe-position, measured-movement, or target
ID controls. Rally, Distract, Break Resolve, and Interrupt retain their distinct
results. No page fork, saved runtime state, numeric cost/cooldown, defence
mechanic, or combat-runtime behavior is added.

Appeal filters to SECURE_WILLING_COOPERATION for Persuasion / One Target or
Small Group at all four Impacts. The family is Help and Counter-ineligible. Its working v0 name
remains eligible for the naming pass. The Builder derives all Method details,
variant outcomes, and eligibility from registries; no page-specific fork is
required.

Rally filters to ESTABLISH_SHARED_RESOLVE for Persuasion / Small Group at all
four Impacts. Establish Shared Resolve is the working v0 name, Help, and
Counter-ineligible. It is the first approved Small Group contract. Large Group
and Faction / Army remain unproven and have no approved variants. Exact variants:

- Minor / ESTABLISH_SHARED_RESOLVE_MINOR: "the selected group steadies around
  one simple immediate course and sincerely pursues it through the current
  meaningful exchange despite ordinary hesitation, confusion, or pressure"
- Standard / ESTABLISH_SHARED_RESOLVE_STANDARD: "the selected group adopts one
  clear shared course as its immediate priority for the rest of the current
  scene and sincerely pursues it despite meaningful fear, confusion,
  disagreement, or pressure"
- Major / ESTABLISH_SHARED_RESOLVE_MAJOR: "the selected group commits to one
  difficult shared course for the rest of the current scene and sincerely
  pursues it despite serious fear, division, personal cost, or danger unless
  decisive circumstances or narrative resolution make that course no longer
  coherent"
- Legendary / ESTABLISH_SHARED_RESOLVE_LEGENDARY: "the selected group forms one
  defining shared resolve, pledge, or cause whose consequences extend beyond
  the current scene and sincerely upholds it until it is fulfilled or
  narratively resolved"

Small Group authoring means one bounded, directly addressable scene-local group,
not a crowd, separated clusters, organisation, settlement, faction, or army.
Before Difficulty, membership, the exact contract-relevant result, and the way
Method/Theme reach everyone are declared. The GD confirms boundary, access,
perception/comprehension, shared situation, member eligibility, Method,
contract, Impact, Scope, and restriction. Rally retains one bounded Declared
Shared Course; Overawe uses one bounded Declared Opposed Course. Success applies
to every accepted member without per-member rolls,
majority interpretation, exemptions, or hidden partial failure. Membership is
fixed and does not propagate to arrivals, outsiders, replacements, followers,
or intermediaries. Each member retains individual turns, tactics, resources,
identity, values, priorities, and judgement. Individual and whole-group
Narrative Resolution follows identifiable fulfilment, impossibility, or
meaningful fictional change.

The Builder exposes Rally and the contract through existing registries only. It
adds no Declared Shared Course, group-member selector, or numeric group size.
Reconciliation retains the family across its four Impact variants and
non-authoring edits, updates exact outcome/descriptor, and keeps Counter false.
Incompatible Intention, Method, or Scope clears family and Counter. Explicit
Custom Outcomes remain Custom. No numeric costs, cooldowns, Scope multipliers,
or headcounts are added.

Overawe filters to stable ID BREAK_SHARED_RESOLVE for Intimidation / One Target
and Small Group at all four Impacts. Break Resolve is the player-facing working
name, Hinder, and Counter-ineligible. Both Scopes purchase abandonment of one
bounded opposed course; Scope selects singular or accepted-group breadth rather
than creating another family. Exact One Target variants and reserved keys:

- Minor / BREAK_SHARED_RESOLVE_ONE_TARGET_MINOR: "the target breaks off one

  the target may choose another coherent response but does not pursue that

- Standard / BREAK_SHARED_RESOLVE_ONE_TARGET_STANDARD: "the target abandons one

  choose another coherent response but does not pursue that course"
- Major / BREAK_SHARED_RESOLVE_ONE_TARGET_MAJOR: "the target abandons one
  central course of opposition for the rest of the current scene and does not
  resume it despite serious pressure, loyalty, personal cost, or command"
- Legendary / BREAK_SHARED_RESOLVE_ONE_TARGET_LEGENDARY: "the target adopts an
  enduring refusal to pursue one defining course of opposition whose
  consequences extend beyond the current scene and maintains that refusal until
  it is narratively resolved"

Existing Small Group variants remain exact:

- Minor / BREAK_SHARED_RESOLVE_MINOR: "the selected group breaks off one small
  immediate shared course of opposition for the current meaningful exchange;
  each member may choose another coherent response but does not pursue that
  course"
- Standard / BREAK_SHARED_RESOLVE_STANDARD: "the selected group abandons one
  clear shared course of opposition for the rest of the current scene; each
  member may choose another coherent response but does not pursue that course"
- Major / BREAK_SHARED_RESOLVE_MAJOR: "the selected group abandons one central
  shared course of opposition for the rest of the current scene and does not

- Legendary / BREAK_SHARED_RESOLVE_LEGENDARY: "the selected group adopts an

The Declared Opposed Course is runtime-only. One Target identifies the target,
one coherent current or imminent course within its agency, and a plausible
Overawe/Theme basis. The GD rejects impossible, bundled, overbroad,
inaccessible, or incoherent declarations before Difficulty. Success ends that
course without selecting an alternative and cannot be evaded through cosmetic
wording, tactics, route, weapon, motive, or a directed intermediary. The target
retains hostility and genuinely distinct choices; selecting an authority figure
never propagates to followers, subordinates, allies, witnesses, or an
organisation.

Small Group retains exact accepted membership, universal eligibility, one
shared course, no per-member roll, no majority interpretation or hidden
exemption, fixed membership, individual agency, and no propagation to arrivals,
replacements, followers, outsiders, or intermediaries. Narrative Resolution
may affect the selected target, the whole group, or one accepted member only
when fulfilment, impossibility, or identifiable meaningful change genuinely
supports it; plot inconvenience, repeated orders, or renamed tactics do not.

The registry-driven Builder exposes Challenge and Overawe for Intimidation and
Break Resolve for both supported Scopes at all four Impacts without a page fork.
Switching One Target to Small Group or back at the same Impact retains
BREAK_SHARED_RESOLVE and updates its exact outcome/descriptor; Counter remains
false. Self, Large Group, Faction / Army, incompatible Method, or incompatible
Intention clears family and Counter. Name, Theme, Dice Count, and restriction
edits retain it. The Builder adds no target/member selector, Declared Opposed
Course, Recipient Type, stored course/target field, numeric cost/cooldown,
quantified Fear, Control, per-member save, or combat-runtime mechanic. Custom
Outcomes remain Custom, and legacy BREAK_RESOLVE remains Custom Method review.

Distract filters to stable ID DIVERT_IMMEDIATE_ATTENTION for Deception / One
Target or Small Group across all four Impacts. Divert Immediate Attention is Hinder,
Counter-ineligible, and the accepted v0 working name. The existing One Target
outcome/key remains exact: "the target's active attention is diverted for the
current meaningful exchange, creating a brief opening for one declared small
immediate action or development to proceed without that target's deliberate
observation or interference" / DIVERT_IMMEDIATE_ATTENTION.

The Small Group outcome/key is: "every accepted member of the selected group has
their active attention diverted for the current meaningful exchange, creating a
brief opening for one declared small immediate action or development to proceed
without deliberate observation or interference from any accepted member" /
DIVERT_IMMEDIATE_ATTENTION_SMALL_GROUP. Every Impact purchases one bounded
Declared Opening; Scope changes breadth while Impact changes the accepted
opening's narrative weight. Large Group and Faction / Army remain unsupported.

Small Group uses exact fixed membership, one Shared Declared Opening, one
coherent competing focus, universal member eligibility, and one Distract/Theme
expression. The GD narrows, revises, or rejects before Difficulty. Success has
no per-member roll, majority interpretation, hidden exemption, secretly
watchful member, or post-success narrowing. Proceed does not mean succeed:
normal timing, costs, rolls, access, tools, barriers, hazards, other observers,
and capability remain. Targets retain physical presence, passive senses,
