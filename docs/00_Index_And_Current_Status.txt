Current Balance And Tuning State

The current tuning model has three distinct layers:

- Power Tuning: canonical underlying power-cost truth.
- Combat Tuning / Monster Baseline Tuning: non-power monster baseline
  contribution.
- Outcome Normalization: downstream radar interpretation and display.

Anti-drift rule:

- If a problem belongs to the power itself, fix it in Power Tuning.
- If a problem belongs to stripped monster body or non-power baseline, fix it in
  Combat Tuning.
- If a problem belongs to radar display curve, fix it in Outcome Normalization.
- Escalate to code only when the issue is routing law, realization law, or a
  missing implementation seam.

Balance North Star:

- Official balance conclusions require runtime-resolved evidence using
  Balance Environment-authored assets through Combat Lab/runtime resolution.
- Balance Environment is the authored test-data scope, not the simulator.
- Combat Lab/runtime is the rules-resolution engine.
- Synthetic fixtures, spreadsheet estimates, simplified probes, and
  non-runtime calculations are diagnostic only unless explicitly routed through
  the live adapter/runtime path and labelled with full provenance.
- Current sequence: attack string calibration first, defence string calibration
  second, attack/defence offset third, real role asset tuning fourth, and cost
  calibration last. Cost values follow runtime truth.
- Do not tune real role assets before attack and defence strings are calibrated
  unless the Lead Designer explicitly approves an experimental value pass.

Current defensive-pool pricing work should not begin from raw win rate alone.
Layer A deterministic canaries must protect mechanics first, Layer B synthetic
batches are targeted diagnostics, and runtime-resolved real authored scenarios
remain primary table-truth balance evidence.

Official balance conclusions must be based on runtime evidence using Balance
Environment-authored assets. Balance Environment data alone is not enough:
reports must identify assetSource, campaignId/campaignName, adapter or
live-loader path, exact command, commit, dirty/clean state, and any unsupported,
ignored, stripped, or approximated mechanics. Reports that bypass Combat
Lab/runtime resolution, or rely on unsupported adapter coverage for material
mechanics, remain diagnostic only for that scope.

Serious numeric balance tuning is additionally gated by formula architecture.
The current formula work has corrected Monster Outcome Calculator Phase 1
expected output for supported natural/equipped attacks, but packet-level power
expected output still depends on callers supplying packet data, and structured
trait reliability, Forge enforcement, and final scenario evidence remain open.
Balance Environment assets are currently useful for data-path testing but are
not final balance evidence until campaign-scoped runtime runs validate the
current formula/runtime state and report adapter coverage.

The balance-environment-core scenarioMatrix preset exists for tiny
campaign-scoped smoke runs against live-authored Balance Environment assets.
Use it to confirm hydration, action sources, unsupported notes, and fallback
count before running medium baselines.

Roleplay Outcome Contract Family Status

LOCKED: a standard Outcome Contract may contain multiple explicitly authored
variants selected by Intention, Method, Scene Impact, and Scope. Each legal
variant owns its exact Success Outcome, Counter eligibility, examples,
exclusions, and future privilege-cost adjustment. A higher Scene Impact never
generically scales outcome prose; every supported tier must be deliberately
authored. Use one stable family ID when the same recognisable privilege changes
in persistence or scale.

LOCKED: DRAW_HOSTILE_ATTENTION is the first multi-variant family. It is the
Hinder contract named Draw Hostile Attention for Intimidation / Challenge and
One Target, with explicit Minor, Standard, Major, and Legendary variants.
Counter is unavailable at every current tier. The family redirects existing
hostility toward the Ability user; it neither creates hostility nor compels a
hostile act. See the locked rules, costing, builder, and launch-roadmap sources
for the exact variant wording and intrinsic boundaries.


Structured Roleplay Method Status

LOCKED: Specific is removed from the active Roleplay Ability model. Intention
defines the broad lane; Method defines the structured legal approach; Narrative
Theme is required player-authored text explaining how this character performs
the Method; Outcome Contract defines the guaranteed result; Scene Impact and
Scope define approved magnitude/persistence and breadth.

"Method defines the approach. Theme personalises the approach. The Outcome
Contract defines the result."

The standard Method registry contains APPEAL, RESCUE, INTERRUPT, CHALLENGE, and
DISCERN_TRUTH. Appeal is the approved Persuasion Method and maps to the locked
Help / One Target family SECURE_WILLING_COOPERATION at all four Impact tiers.
Discern Truth has the locked standard Help
contract family UNCOVER_CONCEALED_TRUTH for One Target at Minor, Standard,
Major, and Legendary Impact. Counter is unavailable at every tier. The family
guarantees reliable concealed information, including a reliable no-qualifying-
truth result; it never compels speech or deliberate revelation. Legacy Specific
values may migrate into Methods, but Specific is no longer player-facing or
normalized runtime state. General Custom Outcome review remains available for
requests outside the standard registry and is never silently migrated merely
because a later standard contract exists.


Roleplay Declaration And Narrative Resolution Status

LOCKED: "Incarnate does not eliminate Game Director judgement. It defines where
judgement belongs and protects the mechanical outcomes players have purchased."
Before Attribute or Difficulty is chosen, the player identifies the Ability and
target, states the runtime Declared Aim, and explains the current Method and
Narrative Theme expression. The GD judges eligibility before the roll; an
accepted successful Outcome Contract is binding and cannot later be weakened,
replaced, evaded, or withheld.

Declared Aim is scene context, not saved Character Builder state. Ongoing
narrative outcomes end only through fulfilment, genuine impossibility, or an
identifiable meaningful fictional change that makes continuation incoherent;
mere inconvenience to the GD, plot, or encounter is insufficient.
